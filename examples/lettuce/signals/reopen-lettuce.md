# Operator reopened Lettuce

The operator reopened Lettuce after noticing a familiar pattern: promising workflow ideas can get overthought and abandoned before they get a fair shot.

The decision is to focus on Lettuce only for now and abandon Agent Action Review and appointment-recovery lanes entirely. Priority is dogfooding today, not polishing a broad platform.

The product wedge should stay solo-operator first: Lettuce watches signal the operator and their agent already create, runs it through explicit lenses, routes what matters into memory, project artifacts, queue previews, and operator notifications, then makes the flow inspectable.

This is not a generic memory app, company brain, or dashboard-first product. The value is lenses before routers: deciding what matters and where it should go before downstream agents act.

First demo should process this Lettuce-reopening signal and produce inspectable packet, notification, JSON, and feed artifacts under `.orbital/lettuce` with no external writes.

Public proof may matter later: if the loop works, it can become a building-in-public artifact showing an operator and agent using Lettuce to improve their operating context.
