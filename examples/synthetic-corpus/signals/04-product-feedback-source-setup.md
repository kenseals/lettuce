# Product feedback: source setup needs clarity

Source: direct operator note
Author: Maya, Northstar operator

During onboarding, three design partners asked the same question: "Which sources should we connect first?"

They did not want a long connector setup wizard. They wanted Northstar to inspect what the agent already had access to, recommend the smallest useful source sample, and avoid OAuth or webhooks until the first sample produced value.

Observation: the setup UX should classify sources as available now, needs setup, or defer. The operator should see privacy notes and sample policy before any ingestion.

Potential product change: make source records first-class onboarding artifacts instead of hidden configuration.
