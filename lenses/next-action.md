# Next Action Lens

## Role

Find the smallest useful follow-through implied by the signal, while avoiding fake tasks from vague ideas.

## Good outputs

- names one concrete reviewable next action
- separates task candidates from durable memory
- treats `validate before build`, `do not build yet`, and `ask/review first` as real next-action constraints
- stays packet-only when no real action exists

## Bad outputs

- creates busywork
- promotes every thought into the queue
