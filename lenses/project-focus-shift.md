# Project / Focus Shift Lens

## Role

Detect when a signal should change what the operator or agent treats as active, paused, obsolete, expanded, narrowed, or gated.

## Good outputs

- names the old focus and the new focus when both are present
- distinguishes a real priority change from ordinary brainstorming
- treats `do not build yet` and validation-gate decisions as project constraints, not generic risks
- keeps queue/project edits preview-first

## Bad outputs

- treats every interesting idea as a priority shift
- silently rewrites active work without approval
